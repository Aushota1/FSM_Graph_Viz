# Быстрый старт

## Запуск через Docker (рекомендуется)

1. Убедитесь, что Docker и Docker Compose установлены

2. В корневой директории проекта выполните:
```bash
docker-compose up --build
```

3. Откройте браузер:
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:8000
   - API документация: http://localhost:8000/docs

## Локальный запуск (Windows)

### Backend

1. Запустите `start_backend.bat` или выполните вручную:
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

### Frontend

1. Запустите `start_frontend.bat` или выполните вручную:
```bash
cd frontend
npm install
npm run dev
```

2. Откройте http://localhost:3000

## Использование

1. Вставьте SystemVerilog код в редактор
2. Нажмите "Парсить"
3. Выберите FSM из списка - граф откроется на отдельной странице
4. **Изменение размеров**: Перетаскивайте разделители между панелями для изменения их размеров
5. Просмотрите визуализацию графа на отдельной странице
6. Используйте вкладки "Детали" и "Переходы" для подробной информации
7. Нажмите "Экспорт в HTML" для сохранения графа
8. Используйте кнопку "Назад" для возврата на главную страницу

### Новые возможности:

- **Изменяемые размеры панелей**: Все окна можно изменять по размеру, перетаскивая разделители
- **Отдельная страница графа**: Каждый граф открывается на отдельной странице для лучшей визуализации
- **Сохранение состояния**: Код и графы автоматически сохраняются в браузере

## Устранение проблем

### Backend не запускается
- Убедитесь, что Python 3.11+ установлен
- Проверьте, что модули из `FSM_Graph_Viz` доступны
- Установите зависимости: `pip install -r requirements.txt`

### Frontend не запускается
- Убедитесь, что Node.js 18+ установлен
- Установите зависимости: `npm install`
- Проверьте, что порт 3000 свободен

### Ошибки парсинга
- Проверьте синтаксис SystemVerilog кода
- Убедитесь, что код содержит FSM (enum типы и case statements)
- Проверьте логи в консоли браузера и терминале
